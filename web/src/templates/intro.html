1. LSTM 모델

LSTM 모델을 사용한 텍스트 분석 과정


각 과정에 대한 설명
RNN의 장기의존성 문제를 해결한 LSTM 모델을 활용하여 문장의 길이와 상관없이 글의 순서를 고려하기 때문에 문맥을 명확하게 파악 가능





2. CNN 모델

CNN 모델을 사용한 이미지 분석 과정
이미지의 RGB값에 K-means clustering 을 사용하여 이미지를 대표하는 색을 뽑아내고
각 감성 클래스 별 해당 색깔이 나올 확률을 가중치 식으로 변형 후
CNN모델의 최종 레이어에 적용.

Datatset은 6가지 감정으로 분류한 Emotion6 / 8가지 감정으로 분류한 Artphoto를 이용


CNN을 활용한 딥러닝의 장점
- 고차원의 컬러 영상의 저차원으로 줄여야 함
- 저차원으로 줄임에도 불구하고 의미있는 특징에 대하여 강조
- 의미있는 특징에 대하여 사용자의 의견이나 판단이 들어가지 않도록 함
- 영상과 같이 고차원의 데이터이지만 많은 데이터가 redundant한 데이터가 많이 섞여 있는 상황에 효율적

- 음악 추천 등 기능 확장 시 높은 확장성을 가진 객체 관계형 데이터베이스 필요

What is Tensorflow-Serving and Celery?
Tensorflow-serving
다른 애플리케이션에서 Machine Learning Model을 사용할 수 있도록
모델을 배포하거나, 모델 API를 제공하는 시스템

celery
메시지 브로커(message broker: redis)와 python 작업 프로세스를 연결하여
비동기 작업을 수행할 수 있는 시스템을 제공하는 분산 처리 메시지 큐(Message Queue)


Tensorflow-Serving을 사용한 이유
- TensorFlow Serving을 사용하면 동일한 서버 아키텍처와 API를 유지하면서 새로운 알고리즘과 실험을 쉽게 배포 할 수 있음
- 다른 유형의 모델 및 데이터를 쉽게 확장 가능


Celery을 사용한 이유
- python에서 가장 많이 쓰이는 웹 프레임워크인 Django과 연동하는 기능을 공식적으로 지원
